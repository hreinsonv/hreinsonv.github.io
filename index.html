Sudoku Game

My partners and I have a strong passion for puzzle games. With the rising popularity of games like Wordle and Crosswords, we quickly realized that Sudoku would be the perfect project for us. Sudoku strikes the right balance—it’s more challenging than Wordle but not as subjectively complex as Crosswords, fitting perfectly into our area of interest. The challenge of developing Sudoku from scratch was too enticing for us to pass up, making the project even more appealing.

Our Program creates a unique sudoku experience on a classic 9 x 9 board. The user will be given the option to select a difficulty: easy, medium, or hard. Based on the selection a random board will be selected from a large collection. The board will be output to the user with spots to fill in. On that same page the user will have to the option to return to the main menu, clear all filled in spots, and to check their current solutions. The game follows the classic rules of sudoku. Each row must only contain the number 1 - 9 with no duplicates; the same applies to each row and block as well. The check button described previously will check this for the user and provide feedback. When the puzzle is completed, and correct, the check button will notify the user

To assist in a greater usability we decided to implement our Sudoku board using a GUI. We utilized the tools provided in JavaSwing to assist us. It allowed us to create a main menu page, difficulty selection page, and a page to display the 9 x 9 sudoku board with text boxes for the user to enter the values into. To implemented this we used a state Design pattern which made it relatively straight forward. Using these tools made the complexity for the user immensely better rather than displaying in the terminal. In the backend, we use a multitude of inheritance principles to create row, column and block subclasses of our board class to easier compare entries and decide validity.

I played a key role in implementing the core of our Sudoku board. Specifically, I was responsible for a significant portion of the following classes: Simulator, DifficultyState, MainMenuState, SudokuBoardState, Row, Block, and adding to our large collection of boards. Planning out these classes, their methods, and their instance variables in the design.md file made this process very smooth. Our group focused on communication and found paired-programming proved to be very helpful. We could exchange immediate feedback resolve issues more quickly than coding individually View our code here: https://github.com/usd-cs/comp305-final-project-sp24-s01s02-hreinsonmikey
